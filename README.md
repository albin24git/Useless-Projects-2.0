### Project Description
NameAI is a playful, AI-powered web application designed to generate a unique name that matches a person's "vibe." Users can capture a live photo and choose between two modes: "Who Am I?" to get a name for themselves, or "Who For Me?" to get a name for an imagined ideal partner.

# The Problem: 
I found out that people don't have a name or even know their father's name. Also, they can't even know the good name of their partner.

# The Solution:
As a solution, I made a web application which will capture our face and give us names, and even names for our father-can find the names for any number of fathers😜- and the attractive feature to find the love of your life.
Basically, the home page has two pathways- (A) "who am i" and (B) "who for me" . 
Under the 'who am i' path, the person's live photo is captured, and then the AI generates a cool name for the person who doesn't remember his/her name. At this stage, the person is again offered an opportunity to know his/her father's name (any number of times) or just collect the new name and leave🏃‍♂️.
Whereas, under the 'who for me' path, the person's live photo will be captured and the highly confidential🫢 real name of the persons life artner would be revealed🫣. Here, if the person likes the partner, they can send a proposal option (you may click it and see how the process happens very efficiently)😏. And wait- if you just don't like the partner- we got the option to generate a new soulmate(until you find the perfect one)😌
To feel the usefulness of my project in depth, just check it out.

### Technical Details
This project was built using a modern web stack, combining a powerful Python backend for AI processing with a dynamic, vanilla JavaScript frontend for a smooth user experience.

# Languages Used:
-Python: For the server-side logic, handling API requests, and interacting with the generative AI.
-JavaScript (ES6+): For all client-side interactivity, including camera control, DOM manipulation, and asynchronous communication with the backend.
-HTML5: For the core structure of the single-page application.
-CSS3: For styling, professional animations, and responsive layout using modern features like Flexbox and CSS Variables.
# Frameworks Used:
-Flask: A lightweight and flexible Python web framework used to create the backend server and API endpoints.
# Libraries Used:
-google-generativeai: The official Google Python client library for accessing the Gemini family of AI models.
-Pillow (PIL Fork): Used in the Python backend to handle and process image data received from the frontend.
# APIs & Tools:
-Google Gemini Pro API: The core generative AI service (specifically the gemini-1.5-flash-latest model) used for analyzing images and generating creative text responses.
-WebRTC (navigator.mediaDevices.getUserMedia): The browser API used to access the user's live camera feed.
-Git & GitHub: For version control and source code management.
-pip & venv: For Python package and virtual environment management.
-Visual Studio Code: As the primary code editor and development environment.
